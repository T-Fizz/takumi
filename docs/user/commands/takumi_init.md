## takumi init

Initialize a new package or workspace

### Synopsis

Initialize a Takumi package in the current directory, or scaffold a new package
in a subdirectory if a name is provided.

If no .takumi/ workspace marker exists above the target directory, a new workspace
is also created (takumi.yaml + .takumi/ directory).

Use --root <project-name> to create a project directory, initialize a workspace
inside it, and optionally scaffold a package with [name].

```
takumi init [name] [flags]
```

### Options

```
      --agent string   AI agent to configure (claude, cursor, copilot, windsurf, cline, none)
  -h, --help           help for init
      --root string    Create a project directory and initialize workspace inside it
```
